# US-EXBOT-001: Start a new ExBot

## Story Statement

As a **USDC Investor with an active Hyperliquid account**,
I want to **start an ExBot that automatically manages a delta-hedged LP position on USDC/ETH**,
So that **my capital earns Uniswap V3 LP fees while the hedge leg suppresses directional ETH loss**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-001-1: Happy path — preflight passes, bot starts successfully
  Given the investor has no existing active/paused/closing/safe_mode/error ExBot
    And the investor's HL isolated margin balance ≥ required margin × 2.0
    And the investor's HL agent key status is "approved" and not expired
    And builder fee (5bps) approval is confirmed
  When the investor submits a start request via the POOL UI
  Then the system creates a bot record with lifecycle_state='preflight'
    And progresses through lp_opening → lp_opened → hedge_pre_open → hedge_post_confirmed → stop_placing → stop_verified → active
    And the POOL UI displays the bot status as "Active" with LP range and hedge size visible

AC-EXBOT-001-2: Edge case — one-bot policy blocks second start
  Given the investor already has one ExBot with status='active'
  When the investor attempts to start a second ExBot
  Then the system rejects the request with message "You already have an active ExBot. Close or wait for the existing bot to finish before starting a new one."
    And no new bot record is created

AC-EXBOT-001-3: Negative path — insufficient HL margin blocks start
  Given the investor's HL isolated margin balance is $300
    And the required margin (including 2.0× buffer) is $700
  When the investor submits a start request
  Then the system blocks the start with message "Required HL margin: $700 (with 100% buffer). Current: $300. Please deposit an additional $400 to HL."
    And no bot record is created

AC-EXBOT-001-4: Negative path — agent key not approved
  Given the investor's HL agent key has status='pending'
  When the investor submits a start request
  Then the system blocks the start with message indicating the agent key must be approved before starting
```

Phase A only: 1 active ExBot per user. `status IN ('active','paused','closing','safe_mode','error')` all count toward the 1-bot limit. The `cooldown` and `parked` lifecycle states map to `status='active'` so are also covered.
