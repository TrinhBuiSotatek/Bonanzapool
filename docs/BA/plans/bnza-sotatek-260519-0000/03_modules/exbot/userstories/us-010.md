# US-EXBOT-010: Handle margin warning and SAFE_MODE entry

## Story Statement

As the **ExBot System Operator (deep-audit and hedge-sync workers)**,
I want to **detect HL margin deterioration and escalate through warning → critical → SAFE_MODE with appropriate restrictions at each level**,
So that **bot positions are protected from liquidation and investors are prompted to add margin before damage occurs**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-010-1: Happy path — margin warning detected, size-increase disabled
  Given a deep-audit fetches HL marginSummary for a bot
    And marginUsage = 0.62 (≥ 0.55, < 0.75 = warning band)
  When the worker evaluates margin status
  Then hedge_legs.margin_status is updated to 'warning'
    And a notification is enqueued: "Margin warning for your ExBot. Consider depositing additional margin to HL."
    And subsequent hedge-sync for this bot only allows size-reduction adjustments (increase direction blocked)
    And the bot status remains 'active'

AC-EXBOT-010-2: Negative path — margin critical triggers SAFE_MODE
  Given hedge_legs.margin_status='warning' and deep-audit detects marginUsage = 0.77 (≥ 0.75)
    And this is the second consecutive critical reading (false-positive prevention)
  When the worker evaluates margin status
  Then hedge_legs.margin_status is updated to 'critical'
    And bot.status transitions to 'safe_mode'
    And all hedge mutations are blocked (no hedge-sync, no LP rebalance, no leverage change)
    And both the investor and admin receive alerts: "Bot entered Safe Mode — Margin critical. Please deposit additional margin to HL."

AC-EXBOT-010-3: Happy path — SAFE_MODE auto-recovery when margin restored
  Given bot.status='safe_mode' (margin-originated)
  When the deep-audit confirms marginUsage < 0.55 AND 3 consecutive successful reconciles pass
  Then hedge_legs.margin_status transitions back to 'ok'
    And bot.status transitions from 'safe_mode' to 'active'
    And the investor receives a notification "Bot recovered from Safe Mode. Monitoring resumed."

AC-EXBOT-010-4: Edge case — effective_leverage > 4.5 triggers SAFE_MODE immediately
  Given a deep-audit computes effective_leverage = 4.8 (exceeds 4.5 threshold)
  When the worker evaluates the leverage condition
  Then bot.status immediately transitions to 'safe_mode'
    And hedge_legs.margin_status is set to 'critical'
    And admin is alerted with exact effective_leverage value and bot_id
```
