# US-EXBOT-003: Pause and resume ExBot

## Story Statement

As a **USDC Investor with an active ExBot**,
I want to **pause my bot temporarily without closing positions**,
So that **I can halt new hedge adjustments while keeping my LP and hedge intact during short-term market uncertainty**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-003-1: Happy path — pause active bot
  Given the investor has an ExBot with status='active'
  When the investor clicks "Pause Bot" and confirms the action
  Then the system sets bot status to 'paused'
    And the UI displays "Paused — Hedge is maintained, LP is maintained"
    And no new light-check or hedge-sync is triggered for this bot
    And the existing HL short position and LP NFT remain untouched

AC-EXBOT-003-2: Happy path — resume paused bot
  Given the investor has an ExBot with status='paused'
  When the investor clicks "Resume Bot"
  Then the system restores bot status to 'active'
    And the UI confirms "Bot resumed. Monitoring resumed."
    And the next light-check is scheduled within 5 minutes

AC-EXBOT-003-3: Edge case — deep-audit continues during pause
  Given the investor's bot is in status='paused'
  When the scheduled deep-audit time arrives (every 6h)
  Then the system runs the deep-audit worker for this bot
    And any anomalies detected are reported to the admin notification queue

AC-EXBOT-003-4: Negative path — pause unavailable in SAFE_MODE
  Given the investor's bot is in status='safe_mode'
  When the investor attempts to pause the bot
  Then the system rejects the action with message "Bot is already in Safe Mode. You can close the bot instead."
```

## Notes

Pause ≠ Close. Pause keeps lifecycle_state unchanged; only `bots.status` switches to 'paused'. The UI button label must read "Pause Bot (hedge is maintained)" — not just "Pause".
