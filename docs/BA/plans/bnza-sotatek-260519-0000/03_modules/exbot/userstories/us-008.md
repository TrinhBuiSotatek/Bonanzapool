# US-EXBOT-008: Manage circuit breaker state per hedge leg

## Story Statement

As the **ExBot System Operator (hedge-sync and light-check workers)**,
I want to **automatically open, half-open, and close the circuit breaker based on consecutive hedge failures — and allow one probe attempt before fully recovering**,
So that **a persistently failing hedge leg does not cause runaway retry loops while still enabling autonomous recovery**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-008-1: Happy path — circuit opens after 3 consecutive failures
  Given a hedge leg has had 2 previous failed hedge-sync attempts within the last 24h
  When a third hedge-sync attempt fails for the same hedge_leg_id
  Then circuit_breakers.state transitions to 'open'
    And circuit_breakers.reset_at is set to now + 1 hour
    And subsequent light-check passes do NOT enqueue hedge-sync for this leg
    And stop monitoring (price-near-stop-audit) continues unaffected

AC-EXBOT-008-2: Edge case — circuit transitions to half_open after reset_at
  Given circuit_breakers.state='open' and reset_at has been reached
  When the light-check worker evaluates this bot
  Then the worker calls allowHedgeSync and receives true (half_open probe claimed atomically)
    And exactly one hedge-sync message is enqueued for this bot (the probe)
    And half_open_probe_used is set to 1 (further probes blocked until state resolves)

AC-EXBOT-008-3: Happy path — circuit closes after successful probe
  Given circuit_breakers.state='half_open' and a probe hedge-sync is in progress
  When the probe hedge-sync completes successfully (reconcile confirms expected size)
  Then circuit_breakers.state transitions to 'closed'
    And failure_count is reset to 0
    And subsequent light-checks resume normal hedge-sync enqueue

AC-EXBOT-008-4: Negative path — probe fails, circuit re-opens
  Given circuit_breakers.state='half_open' and the probe hedge-sync fails
  When the failure is recorded
  Then circuit_breakers.state transitions back to 'open'
    And reset_at is reset to now + 1 hour
    And admin receives a notification "Circuit re-opened after failed probe for bot {id}"
```
