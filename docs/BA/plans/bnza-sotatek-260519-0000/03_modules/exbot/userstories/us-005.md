# US-EXBOT-005: Run periodic light-check without HL calls

## Story Statement

As the **ExBot System Operator (automated worker)**,
I want to **evaluate each active bot's rebalance need every 5 minutes using only D1 and MarketDataDO — without calling Hyperliquid**,
So that **10,000 bots can be scanned within a 5-minute window without breaching the HL rate limit of 1,200 weight/min**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-005-1: Happy path — light-check detects drift and enqueues hedge-sync
  Given an active bot whose |targetShortEth - lastKnownShortEth| × uniPoolPrice > $25
  When the light-check worker processes this bot
  Then the worker evaluates RebalanceReason[] using only D1 state and MarketDataDO (zero HL API calls)
    And enqueues a hedge-sync message with reasons=['drift_threshold'] and stateVersion
    And updates next_light_check_at = now + 5min + jitter(±45s)

AC-EXBOT-005-2: Edge case — stop trigger detected, routes to stop-audit not hedge-sync
  Given an active bot where markPriceUsd >= hedge_legs.stop_price
  When the light-check worker processes this bot
  Then the worker sets hedge_legs.stop_trigger_crossed_at (only if currently NULL)
    And enqueues a price-near-stop-audit message
    And does NOT enqueue a hedge-sync message for this trigger

AC-EXBOT-005-3: Edge case — circuit breaker open, hedge-sync suppressed
  Given a bot where circuit_breakers.state='open'
  When the light-check worker processes this bot
  Then the worker does NOT enqueue a hedge-sync message
    And still evaluates and enqueues price-near-stop-audit if stop criteria met
    And continues normally (does not skip the bot entirely)

AC-EXBOT-005-4: Negative path — bot in lp_rebalancing or lp_closing skipped
  Given a bot with lifecycle_state='lp_rebalancing'
  When the light-check worker encounters this bot
  Then the worker skips all rebalance evaluation and enqueuing for this bot
    And moves to the next bot without error
```
