# US-EXBOT-012: Admin force-close an ExBot

## Story Statement

As the **ExBot Admin (zen)**,
I want to **force-close any ExBot from the admin dashboard regardless of its current lifecycle state**,
So that **compliance requirements, security incidents, or unrecoverable system states can be resolved without investor action**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-012-1: Happy path — admin force-close triggers bot_safe_close flow
  Given the admin selects an active bot in the admin dashboard and clicks "Force Close"
  When the admin confirms the action with a reason
  Then the system initiates bot_safe_close (hedge-first close)
    And bot.status transitions to 'closing'
    And the HL hedge is fully closed (reconcile confirms size = 0)
    And BnzaExVault.vaultClose is called, USDC parked into uninvested_balances
    And close_operations record tracks full state transition to 'done'
    And the investor receives notification "Your ExBot was administratively closed. USDC is available for withdrawal."

AC-EXBOT-012-2: Edge case — force-close on a bot already in SAFE_MODE
  Given the admin selects a bot with status='safe_mode'
  When the admin initiates force close
  Then the system proceeds with bot_safe_close using the existing hedge state
    And if the HL hedge cannot be closed (HL unreachable), creates a residual_hl_liability record
    And admin is notified of the outstanding liability with the exact amount

AC-EXBOT-012-3: Edge case — emergency multi-sig close when contract-level access needed
  Given a BnzaExVault contract bug is detected and the operator key cannot safely interact
  When the multi-sig admin uses the emergency function
  Then BnzaExVault transfers the LP NFT directly to the investor's wallet via emergencyTransfer
    And bot.status is set to 'error' by the Operator
    And admin receives a task: "Close HL hedge manually for bot {id} — agent key valid: {Y/N}"

AC-EXBOT-012-4: Negative path — force-close attempt on already-closed bot
  Given the bot has lifecycle_state='closed'
  When the admin attempts a force-close
  Then the system rejects with message "Bot {id} is already closed. No action needed."
    And no close_operations record is created
```
