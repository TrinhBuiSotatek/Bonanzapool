# US-EXBOT-004: Close ExBot and redeem funds

## Story Statement

As a **USDC Investor who wants to exit their ExBot position**,
I want to **close my ExBot and receive my USDC back**,
So that **I can fully exit the strategy and retrieve my capital without manual on-chain steps**.

**INVEST check:** I✓ N✓ V✓ E✓ S✓ T✓

## Acceptance Criteria

```gherkin
AC-EXBOT-004-1: Happy path — user_redeem flow completes within SLA
  Given the investor has an active ExBot
  When the investor submits a "Close Bot" request and confirms "close everything, return USDC"
  Then the system triggers on-chain BnzaExVault.redeem(tokenId), liquidating the LP instantly
    And the LP-portion USDC is returned to the investor's wallet in the same on-chain transaction
    And the system enqueues a user_redeem job (highest priority) to close the HL hedge
    And within 5 minutes of the redeem event detection, the HL short is fully closed
    And the HL-portion USDC is sent to the investor after hedge close is confirmed
    And the bot lifecycle_state transitions to 'closed'

AC-EXBOT-004-2: Edge case — hedge close takes longer than SLA
  Given a user_redeem hedge close job is processing
  When 5 minutes elapse from the redeem event without confirmed hedge closure
  Then the system escalates the job to admin alert with "user_redeem hedge SLA breached for bot {id}"
    And the LP-portion repayment is NOT reverted (it was already returned in the redeem tx)
    And the admin receives a residual_hl_liability record to handle manually

AC-EXBOT-004-3: Negative path — hedge close fails after LP already liquidated
  Given the LP-portion USDC was already returned to the investor in the redeem tx
  When the HL hedge close fails due to HL API unavailability
  Then the system marks the close_operations record as 'residual_hl_liability'
    And notifies admin with exact bot_id, hedge size, and estimated USD value outstanding
    And does NOT attempt to reverse the LP-portion repayment
```
